# Lumina System UI Critique

Design Health Score: 28/40

Scope: Git Assistant and DevDock workbench UI. Evidence: source review, prior user screenshots in this thread, and impeccable detector output.

Anti-pattern verdict: The UI does not immediately read as AI-generated. It has a restrained desktop-tool tone, stable panel rhythm, and a sensible shift toward a workbench model. The remaining generic feel comes from repeated bordered surfaces, similar button weights, and dense local patterns that are not yet consolidated into a shared design system.

What works:
- The Git/DevDock top switch is now conceptually correct and visually stable.
- DevDock's collapsible multi-project list plus right process panel is the right information architecture.
- The interface uses compact developer-tool density without drifting into a marketing/dashboard aesthetic.

Priority issues:
1. P1: Component vocabulary is still duplicated across Git and DevDock. Similar concepts such as toolbar tags, pills, drawers, buttons, panel headers, and rows are implemented locally, so consistency can regress easily.
2. P1: Git Assistant still has high cognitive load. The commit editor, AI model controls, remote tools, log/history/review actions, and change table compete for attention.
3. P1: Runtime workflows need stronger feedback and recovery. Process launch, port discovery, logs, exited states, and action availability need consistent status semantics.
4. P2: Accessibility is partially covered but not systematic. Some state changes are visual-first, logs are not live regions, and dense rows need stronger focus/keyboard affordances.
5. P2: Responsive behavior is desktop-first with fixed side panels. That is acceptable for a Tauri tool, but narrow-window handling should be explicitly designed.

Recommended next work:
- Extract shared workbench primitives: topbar, segmented switch, toolbar tag, panel header, row, drawer, status pill, primary/secondary/danger buttons.
- Distill the Git Assistant right panel into primary action plus grouped secondary actions.
- Harden process and log states with consistent running/starting/exited/error semantics.
- Add an accessibility pass for focus states, ARIA live output, keyboard paths, and non-color status labels.
